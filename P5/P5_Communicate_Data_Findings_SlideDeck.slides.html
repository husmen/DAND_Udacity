Data Exploration on Kaggle's House Prices Dataset¶

by Houssem Menhour¶

Investigation Overview¶

In this presentation, I'll be showcasing some key findings from my EDA process on the dataset described bellow.

My main interest is finding out what features of a house decide its pricing.

Dataset Overview¶

This is a popular dataset from Kaggle with historical sale prices of houses and their features or characteristics. It is meant for practicing regression techniques and predicting sale prices, a task that requires EDA first.

This dataset comes in 1460 entries and 81 columns, 38 of which are numerical while 43 are categorical. Some of the categorical columns represent ordinal values, this mainly applies to variables of a quality related value, while others are nominal.

The full data and its description is available here.

Distribution of Sale Prices¶

The original sale prices are skewed, we need to transform them to make it closer to a normal distribution.

Applying np.log1p significally reduces the skew value from 1.88 to 0.12.

The same transform will be applied to the other skewed numerical variables.

In [14]:
skewness1 = df2['SalePrice'].skew()
skewness2 = np.log1p(df2['SalePrice']).skew()


f = plt.figure(figsize=(12, 4))
gs = f.add_gridspec(1, 2)

ax = f.add_subplot(gs[0,0])
ax = sns.distplot(df2['SalePrice'], color="b", kde=True);
ax.set(title="(skew = {})".format(round(skewness1,2)))

ax = f.add_subplot(gs[0,1])
ax = sns.distplot(np.log1p(df2['SalePrice']), color="b");
ax.set(title="(skew = {})".format(round(skewness2,2)))

f.suptitle('Sale Price distribution', y = 1.03, fontsize=20)
f.tight_layout()
